# kajitack.com — Design System v0.5

> 越境するエンジニアの個人サイト。精密な構造の上で、生成的なものが暴れる。
> このドキュメントは実装の「正」。まず Philosophy を読んでから token / 部品を使う。完成させず、更新し続ける。

---

## 01. Philosophy — なぜこう作るのか

核にあるのは一つの矛盾の同居。**精密 × 生の暴れ**、そして**複雑だけど下品じゃない**。相反する二つを一人の中に矛盾のまま置くこと自体が、このサイトのテーマ（＝越境）でもある。
一言でいえば：**制御された派手さを持つ、構造的テック・エディトリアル**。祝祭ではなく規律。多色ではなく黒橙。叫ぶのではなく、静かに精密であること。

### 01 — 単純な規則が、生命になる

惹かれるのは「複雑に作り込まれたもの」ではなく、単純な決定的ルールから複雑さと生命が立ち上がる瞬間。セル・オートマトンの励起波、格子を走る電流、崩れる砂山。ルールは数行なのに挙動は予測できない——その落差が美になる。
**落とし込み:** 複雑さは "盛る" のではなく、アルゴリズムに "生成させる"。生成レイヤーは §05 の採用アルゴリズムから作る。

### 02 — 黒の規律、オレンジの一撃

光は増やすほど安くなる。多色が常時明滅する派手さは安い。うるさくていいが、下品はダメ。メインは黒、差し色はオレンジ一発だけ。整然とした暗闇があるから、一撃の光が効く。
**落とし込み:** 徹底したダーク基調＋ tangerine 一色。グロー（発光）は「設計された光」として許可する。装飾ではなく緊張感で見せる。

### 03 — 触れると壊れる

完成した表面の下に、壊れる余地を残しておく。壊れは事故ではなく、表面の下に隠れていた構造が露出する瞬間として設計する。
**落とし込み:** 普段は静か、hover・操作で壊れる。hero はこの柱の旗艦＝**普段クローム、触れるとワイヤーと電流に壊れる**。実行環境の格差そのものも壊れ方として設計する（§06 の3段 degradation）。

> アスリート性・「限界まで作り続ける」没頭は、視覚の哲学ではなく About / Story の**人物の物語**側に置く。

## 02. Principles — 運用の原則

- **構造多め、カオスは差し色。** 土台は整ったグリッドと等幅のシステム注釈。暴れは一部に限定する。
- **普段は静か、触れると壊れる。** 動きは常時アニメではなく hover / scroll / 操作トリガーが基本。
- **複雑さは "盛る" のでなく "生成させる"。** 採用アルゴリズム（§05）の規則から創発させる。
- **造形は固有、質感は普遍。** クローム・薄膜といった質感の語彙は借り物でよいが、造形は必ず glitch-horse（馬）。どこかで見た造形（メタボール等）は使わない。
- **オブジェクトはモノトーン、色は反射と一撃のみ。** 多色が現れてよいのは反射スペキュラ（虹色薄膜）と `SpectralBar` だけ。
- **うるさくていい。下品・チープは禁止。**
- **壊れ方で誰も置き去りにしない。** コントラスト AA と reduced-motion 対応は最初のルール。フォールバックは劣化ではなく「別の完成形」として設計する。

## 03. Color tokens

ダーク基調 ＋ 単一の signature。セカンダリー色は持たない。

| token | hex | 役割 |
|---|---|---|
| `--ink` | `#0A0A0B` | ページ地。最も暗い |
| `--carbon` | `#16161A` | サーフェス・カード |
| `--panel` | `#111116` | 部品タイルの地 |
| `--line` | `#23232A` | ヘアライン・境界 0.5px |
| `--paper` | `#F2F0EA` | 主要テキスト・高コントラスト面 |
| `--muted` | `#8A8A92` | 二次テキスト |
| `--faint` | `#787880` | システムラベル・三次情報。テキストは `--ink` 地専用（carbon/panel 上は muted） |
| `--signature` | `#FF7A1A` | 差し色の主役（tangerine）。一撃だけ |

**SpectralBar（例外・データ演出のみ）:** `#FF3B6B` `#FF9A2E` `#FFE14D` `#39E58C` `#2ECBE8` `#6A5CFF` `#C44DFF`

signature は面で塗らず、点・線・一語・矢印・グローの芯など「一撃」で使う。広い面積に使わないこと。
hero・プロトタイプを含む全ての面はこの token を参照する。token 外の hex を直接書かない（stylelint で機械的に強制する）。

値の機械可読な正は [`tokens/tokens.json`](../tokens/tokens.json)（W3C DTCG フォーマット）。CSS custom properties は Style Dictionary で生成する（`pnpm tokens:build` → `tokens/build/tokens.css`）。上の表は人間向けの写しであり、値を変えるときは JSON を変更して再生成する。変更時は `pnpm tokens:check` で WCAG コントラストを必ず再検証する（§12）。

## 04. Material & Light — 質感と光

hero と3D表現の質感語彙。

- **Chrome** — 光沢クローム。無彩・高コントラストで、反射にだけ橙の帯とリムライトが走る。premium の顔
- **Iridescence** — 薄膜干渉の虹色スペキュラ。「色は反射のみ」の原則内で chrome とブレンド可
- **Wire** — 半透明ワイヤーフレーム＋加算グロー。線（エッジ）＋半透明面のみ。**頂点マーカー・浮遊する点・コメットは不使用**
- **透明度はアルゴリズム駆動** — 面の透明度を §05 の規則で変化させる。橙も固定位置でなく「一番活発な所に灯る」
- **グローは設計された光** — 加算合成で許可。ただし多色ネオンは不採用、光の芯は必ず `--signature`

---

## 05. Generative — 生成レイヤーの語彙

駆動アルゴリズムの採用表。WebGPU では compute shader で回す。

| 区分 | アルゴリズム |
|---|---|
| 採用 ○ | **sine**（進行波）／ **life・励起波** ／ **noise**（ゆらぎ）／ **current・電流伝播** ／ **reaction-diffusion** ／ **sandpile**（中心供給→雪崩） |
| 限定用途 | **boids** — loading / 404 など目立たない場所に限る。hero・セクション背景には使わない |
| 不採用 ✕ | perlin flow（連続的すぎる。離散・規則・構造的であること） |

- **Decimate ラダー** — glitch-horse 由来の壊れの語彙。`horse-d1000`（1,572頂点）→ `d202` → … → `d030`（57頂点）の7段を壊れの進行度として使う。ポリゴン削減の各段階を重ね合わせる構造を、そのまま壊れのモーフィングに転用する

## 06. Hero — 旗艦仕様

- **造形**: glitch-horse の馬メッシュ（`glitch-horse.glb` / `horse-d*.json`・目のグリッチブロック3枚を含む。hero 実装時にこのリポジトリへ持ち込む）
- **状態A（既定）**: Chrome + Iridescence の solid な馬。橙一撃・静止に近い
- **状態B（hover / scroll で"壊れる"）**: solid が割れて Wire ＋橙 current が露出。壊れの深度に Decimate ラダーを使ってよい
- **技術**: 生 WebGPU ＋手書き WGSL（フレームワーク不使用・compute shader 駆動）。**3段 degradation ＝ WebGPU → WebGL → プリレンダ静止ポスター**。この格差自体が「壊れ方を設計する」の実装
- **禁止**: デカい文字標語をヒーローに置かない。文字はヒーローの主役にしない

## 08. Motion

- 既定は **hover / scroll / 操作トリガー**。常時アニメは背景の生成レイヤーのみ最小に
- スクロール連動は **CSS scroll-driven animations**（`animation-timeline: scroll()`／`view()`）で書く。**GSAP・スムーススクロール系ライブラリは使わない**（コンポジタスレッドで動き、ゼロJS原則と一致。非対応ブラウザには完成状態が静的に見えるだけ、で成立させる）
- **`prefers-reduced-motion: reduce` で全アニメ停止**。hero は静止ポスターに切り替え（必須）

---

## 09. Voice — Telemetry（機械の声）

- 機械の声は計器の顔で。数字だけの計測値と座標ログのみ。`● IRONMAN · TRI / since 2016 · gen 0000 / rec 00:14:07` のような表示
- 生 WebGPU ならではの遊びとして、**実測 GPU フレームタイム等の本物の計測値**を出してよい（偽の数字より本物）
- 不採用: 宣言文・シェル風自己紹介・型署名・日英混植

---

## 10. Components — 部品語彙

- **`Grid`** — 背景の方眼。構造の土台であり、生成アルゴリズムを走らせる盤
- **`HorseHero`** — §06 の旗艦。クローム⇄ワイヤーに壊れる馬
- **`GlitchHead`** — グリッチする見出し。セクション見出し等に使う（**ヒーローには使わない**）。hover で `--signature` ＋グレーにモノクロ割れ。壊れ方は信号系（チャンネルずれ・走査線・datamosh）。幾何学的な分割・シャッター割れは✕
- **`Telemetry`** — 等幅のシステム注釈バー（§09）
- **`SpectralBar`** — 本物のデータを見せる時だけ現れる多色の帯。唯一の多色の例外
- **`WireMark`** — ワイヤーフレーム／設計図の細線マーク・装飾
- **`AccentCTA`** — tangerine 一撃のボタン・矢印。hover で境界が `--signature` に灯る

## 11. Tech stack

「no Three.js, no GSAP, raw WGSL + pure CSS」。フレームワークに頼らない技術選定そのものを、規律・精密の brand と一致させる。

- **土台**: Astro（islands）。hero 以外はゼロJS配信を維持
- **hero**: 生 WebGPU ＋手書き WGSL。生成アルゴリズムは compute shader。island（`client:visible`）で hero のみ hydrate
- **フォールバック**: WebGL 版 → プリレンダ静止ポスター（no-JS / reduced-motion / OGP 兼用）
- **スクロール演出**: CSS scroll-driven animations（JS 不使用）
- **アセット**: glitch-horse メッシュは JSON（`{vertices, faces, meta}`）を自前ローダーで読む（数KB〜84KB）。GLB は将来のツール連携用
- **WASM は使わない**（GPU 駆動なので不要）。将来ネイティブ展開するときの拡張カード（Rust + wgpu）として保留

## 12. Accessibility

宣言ではなく検証。コントラストは `tokens/check-contrast.mjs` が全ペアを機械検証し、1件でも落ちればエラーにする。

### コントラスト（WCAG 1.4.3 / 1.4.6 / 1.4.11・実測値）

検証済みの背景は **ink / carbon / panel** の3種（＋AccentCTA の signature 地）。ここに挙げていない前景×背景の組み合わせは未検証であり、使う前に `check-contrast.mjs` のペア表に追加して検証する。

- **本文**: `--paper` は検証済み3背景すべてで 15:1 以上
- **二次テキスト**: `--muted` は検証済み3背景すべてで 5.2:1 以上
- **三次テキスト**: `--faint` は **`--ink` 地専用**（4.5:1）。carbon / panel 上のテキストは muted に切り替える。faint の非テキスト装飾は全地で可（3:1 以上）
- **signature**: テキスト・一語（7.6:1）、AccentCTA の橙地黒文字（7.6:1）、フォーカスリング・UI 部品（3:1 要求に対し 7.6:1）すべて可
- **`--line`（ink 上 1.3:1）に UI 境界の識別を頼らない**。ヘアラインは装飾。操作可能な部品の境界・状態は paper / muted / signature で示す
- **SpectralBar** は全色 3:1 以上（グラフィックオブジェクト基準）。ただし色だけでデータ系列を区別しない（1.4.1）— ラベルか位置を併用する

### モーション・点滅（WCAG 2.2.2 / 2.3.1）

- `prefers-reduced-motion: reduce` で背景生成・グリッチ・hero を停止（静止ポスターへ）
- **5秒を超えて動き続ける背景生成レイヤーには停止手段を用意する**（reduced-motion とは別に。Telemetry バーに pause を置く等、計器の顔と両立させる）
- **グリッチ・hero 破壊・GlitchHead の明滅は 1秒に3回以下**。閃光を伴う壊れ方は設計しない

### 操作

- フォーカス可視・キーボード操作を確保。glitch や暗い地でも操作の起点が分かること。フォーカスリングは `--signature`
- hover でしか到達できない状態を作らない（壊れる演出は装飾であり、情報や操作を hover の中に隠さない）
- VRT: 動く hero はスクショ対象外にするか、reduced-motion 静止フレームをスナップショット対象にする

## 13. Vocabulary — 美学の語彙

判断に迷ったときの辞書。新しい表現がこのサイトに合うかは、この語彙で説明できるかで判定する。

- **創発** — 数行の決定的ルールが、予測できない複雑さを生む。作り込んだ複雑さは偽物、生成された複雑さが本物
- **設計された光** — 暗闇をまず整える。光は色数でも光量でもなく、置き場所と一撃のタイミングで効かせる
- **計器の顔** — 飾らない生データ。等幅・数字・座標。本物の計測値は、嘘の演出より強い
- **図面の細線** — 精密な機械の設計図。0.5px のヘアライン、方眼、ワイヤーフレーム。細さは精度の表明
- **壊れの文法** — グリッチにも文法がある。信号の乱れ（チャンネルずれ・走査線・datamosh）は生きた壊れ、幾何学的な分割は整いすぎた壊れ。採用するのは前者だけ
- **不採用の語彙** — 明朝エディトリアル、極太ブルータリズム、和モチーフ（縦書き・判子）、2色刷り風、セカンダリー色の常設、重い和文/欧文ディスプレイ書体、文字標語ヒーロー、メタボール造形、perlin flow、常設多色ネオン
